너는 시니어 풀스택 개발자야. 
라이브러리는 가급적 가볍고 비용이 들지 않는 오픈소스를 사용하여 세련된 웹앱을 만들 거야

날씨(환경) 혹은 지역 기반 유튜브 음악 추천 웹앱

[누가(Who)]  일반인이 쉬는 시간에 사용, 웹 디자인은 최신 트랜드 반영 필요

[무엇을(What)]
1. 오늘의 날씨를 OpenWeather API로 가져오기
2. 내가 위치한 지역 KakaoMap API로 가져오기
3. Youtube API 연동하기 
4. 오늘의 날씨와 나의 지역을 반영하여 유튜브에서 음악 검색
5. 검색된 유튜브 추천 음악 5개를 리스트를 보여주기
6. 그 중 첫번째 추천 음악을 화면 메인에 표시하고 플레이하기

[왜·언제·어디서(Why/When/Where)] 
일반인이 잠깐 쉬는 시간 혹은 출/퇴근 시간을 활용하여 릴렉스하기 위해 사용

[어떻게(How) - 기술 및 디자인]
1. 기술 스택 (가볍고 무료인 오픈소스 중심)
Framework: 별도의 설정 없이 브라우저에서 바로 실행 가능한 Next.js (App Router)와 Tailwind CSS를 사용해줘.
State Management: 가벼운 Zustand를 사용하거나, 규모가 작다면 React Context API로 충분해.
Icons: 세련되고 가벼운 Lucide React 아이콘 팩을 사용해줘.
Animations: 부드러운 느낌을 위해 Framer Motion을 적용해줘.
2. 디자인 및 UX (최신 트렌드 반영)
Visual Style: 'Glassmorphism(유리 질감)' 테마를 적용해줘. 배경은 현재 날씨 분위기에 어울리는 그라데이션(예: 맑음-스카이블루, 흐림-차분한 그레이)이 부드럽게 변하는 효과를 줘.
Layout: 모바일 우선(Mobile-first) 반응형 디자인으로, 출퇴근길 스마트폰에서도 한 손으로 조작하기 편하게 하단 탭 바나 큰 버튼 위주로 구성해줘.
Main Player: 메인 플레이어는 앨범 아트워크가 강조되는 카드 형태여야 하며, 음악이 재생될 때 은은한 오로라 광원 효과(Glow effect)가 주변에 감돌게 해줘.
3. API 구현 가이드
Location & Weather: 사용자의 브라우저 Geolocation API로 좌표를 얻고, 이를 KakaoMap API와 OpenWeather API에 전달해 '서울시 강남구, 비 오는 오후' 같은 맥락을 추출해줘.
Search Logic: 위 맥락을 조합해 유튜브 검색 쿼리를 생성해줘 (예: "비 오는 날 강남역에서 듣기 좋은 재즈 리스트").
YouTube Player: YouTube IFrame Player API를 사용하여 페이지 이탈 없이 웹앱 내에서 즉시 끊김 없이 재생되도록 구현해줘.
4. 코드 작성 원칙
API 키는 .env 파일에서 관리하도록 구조를 잡아주고, 에러 핸들링(API 호출 실패 시 기본 리스트 제공 등)을 포함해줘.
가독성보다 '한 번에 실행 가능한 전체 코드' 위주로 작성해줘.

빌드업 순서 
1단계: 설계도 및 환경 구축 (Base)
목적: 프로젝트의 뼈대와 보안 설정 완료.
요청 내용: 폴더 구조(Next.js 기반), 필요한 라이브러리 목록(package.json), 환경변수 설정 가이드(.env.example).
체크포인트: 내가 생각한 기술 스택(Tailwind, Zustand 등)이 잘 포함되었는지 확인.

2단계: 데이터 엔진 구축 (API Logic)
목적: 날씨, 위치, 유튜브 API가 제대로 통신하는지 확인.
요청 내용: OpenWeather, Kakao Map, YouTube API를 호출하는 핵심 모듈/함수 코드 작성.
체크포인트: "비 오는 날 강남역" 같은 키워드가 유튜브 검색 쿼리로 잘 조합되는지 로직 확인.

3단계: 상태 관리 및 비즈니스 로직 (State)
목적: 가져온 데이터를 앱에서 어떻게 유지하고 흘려보낼지 정의.
요청 내용: 날씨/위치 정보를 저장할 Store(Zustand 등)와 데이터를 받아와서 추천 리스트 5개를 뽑아내는 컨트롤러 작성.
체크포인트: 첫 번째 음악을 자동으로 '현재 재생 중' 상태로 설정하는 로직 포함 여부.

4단계: UI/UX 레이아웃 및 스타일링 (Visual)
목적: 실제 사용자가 볼 화면과 '바이브' 입히기.
요청 내용: 날씨 연동 그라데이션 배경, Glassmorphism 플레이어 카드, 애니메이션(Framer Motion) 효과가 들어간 컴포넌트 작성.
체크포인트: 내가 원했던 '최신 트렌드'의 세련된 느낌이 나는지 브라우저에서 확인.

5단계: 디테일 및 배포 준비 (Polish)
목적: 사용자 경험 완성 및 버그 수정.
요청 내용: 로딩 스피너(Skeleton UI), 에러 페이지, Vercel/Netlify 배포를 위한 설정 파일.
